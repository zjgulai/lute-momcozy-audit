⚠ 关键洞察 #1 · 流失断崖在第一节
96.3% 访客在「加购前」就跑了，这才是病灶。
漏斗最大流失发生在 访客 → 加购（损失 96.33% 访客）· 行业平均损失 90%。
Cart Abandonment 仅 28.8%（行业 70%）· 说明 加进购物车的人很坚定，问题不在结账。
→ 优化重点：性能 + UX（让访客走进 PDP / 让 PDP 让人想加购），而不是优化结账流程。
⚠ 关键洞察 #2 · CVR 双口径警示
Excel 自报 CVR 0.88% ≠ 反算 0.28% · 真相是后者
Excel 字段 overall_cvr 0.88% 与 订单数 / 访客数 反算 0.28% 矛盾 3.1 倍。
可能解释：Excel 0.88% 是「加购 → 支付」子漏斗口径（135d 累计 7.23M 加购 × ~7.6% = 547.8K 成交 ≈ 0.88%）。
工业标准 CVR = 订单 / 访客 = 0.28%（行业基线 1.5-2.5% 的 11-19%）· 实际比 Excel 上报更糟。
→ 字典里此处 双口径都报，路特可与 Marketing/Analytics 团队对齐口径。
